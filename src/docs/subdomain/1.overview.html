DNS Records ที่รองรับ

สามารถเพิ่ม record ได้หลายประเภทในโดเมนเดียวกัน โดยมีข้อจำกัดบางอย่างตามมาตรฐาน DNS

CNAME
รองรับ
ชี้ไปยัง hostname เช่น Vercel, Netlify
A Record
รองรับ
ชี้ไปยัง IPv4 address
AAAA Record
รองรับ
ชี้ไปยัง IPv6 address
TXT Record
รองรับ
สำหรับ domain verification
MX Record
รองรับ
สำหรับ email routing

ข้อจำกัด: CNAME ไม่สามารถอยู่ร่วมกับ A, AAAA หรือ MX ในชื่อเดียวกันได้ ตามมาตรฐาน RFC 1912 ; ระบบจะแจ้งเตือนอัตโนมัติเมื่อเกิด conflict

ข้อกำหนดสำคัญ (สรุปย่อ)

รายละเอียดเต็มอยู่ใน ข้อกำหนดการใช้งาน ด้านล่างนี้คือสิ่งที่ควรรู้ก่อนสมัคร

  • ห้าม
  • เนื้อหา NSFW หรือเนื้อหาสำหรับผู้ใหญ่ทุกประเภท
  • เนื้อหาที่ผิดกฎหมายไทย หรือละเมิดลิขสิทธิ์
  • Phishing, Malware หรือเว็บไซต์หลอกลวง
  • แก้ไข subdomain ของผู้อื่นโดยไม่ได้รับอนุญาต (ไม่ใช่ owner/co-owner)
  • อนุญาต
  • ใช้งานจากที่ไหนก็ได้ ไม่จำกัดเฉพาะในโรงเรียน
  • เพิ่ม co-owner เพื่อให้ผู้อื่นช่วยจัดการ DNS record ร่วมกัน
  • แก้ไข DNS record ได้ตลอดเวลาผ่านหน้าเว็บหรือ GitHub PR
  • ลบโดเมนของตัวเองได้ทุกเมื่อ

    • maintainer (@aitji) มีสิทธิ์ระงับหรือลบ subdomain ที่พบว่าใช้งานไม่เหมาะสมโดยไม่ต้องแจ้งล่วงหน้า หากพบการใช้งานที่ไม่เหมาะสม สามารถรายงานได้ที่ abuse

คำถามที่พบบ่อย

ใครลงทะเบียนได้บ้าง? นักเรียนและครูโรงเรียนท่าตะโกพิทยาคมทุกคน โดยต้องมีบัญชี GitHub
ไม่มีการตรวจสอบว่าเป็นบุคลากรจริงหรือไม่ในระบบอัตโนมัติ แต่หากพบการใช้งานที่ไม่เหมาะสม maintainer จะดำเนินการตามที่เห็นสมควร

Subdomain มีวันหมดอายุไหม? ไม่มี subdomain จะยังคงใช้งานได้ตราบเท่าที่ไม่ถูกลบหรือระงับ
อย่างไรก็ตาม หากเว็บไซต์ปลายทางไม่ตอบสนองเป็นเวลานาน อาจถูก maintainer ทบทวนเพื่อคืน namespace

สามารถแก้ไข Record หลังลงทะเบียนแล้วได้ไหม? ได้, เข้าไปที่ หน้าจัดการ
แล้วกด "แก้ไข" ที่โดเมนที่ต้องการ ระบบจะสร้าง PR ใหม่เข้า GitHub อัตโนมัติ
การเปลี่ยนแปลงจะมีผลหลัง maintainer approve และ merge

Co-owner ทำอะไรได้บ้าง? Co-owner สามารถเปิด Pull Request เพื่อแก้ไข DNS record ของ subdomain นั้นได้
แต่ไม่สามารถลบโดเมนหรือเปลี่ยน primary owner ได้
GitHub Actions จะตรวจสอบว่าผู้เปิด PR เป็น owner หรือ co-owner ของโดเมนนั้นจริง
*Co-owner และ owner สามารถใช้เว็บไซต์สำหรับแก้ไขได้ที่นี่, หากไม่ต้องการแก้ไขใน GitHub

รองรับ provider อะไรบ้าง? ระบบมี template พร้อมใช้สำหรับ Vercel, GitHub Pages, Netlify, Cloudflare, Render, Railway และ Fly.io
*หากใช้ provider อื่น สามารถเพิ่ม DNS record แบบ manual ได้เช่นกัน

ขอหลาย subdomain ได้ไหม? ได้ ไม่มีการจำกัดจำนวน subdomain ต่อ 1 บัญชี GitHub อย่างไรก็ตามขอให้ใช้งานจริงและไม่จอง namespace โดยไม่มีวัตถุประสงค์

ลบโดเมนยังไง? เข้าไปที่ หน้าจัดการ แล้วกดปุ่ม "ลบ" ที่โดเมนที่ต้องการ ระบบจะส่ง request ลบ DNS record ออกจาก Cloudflare และ repository ทันที
