อะไรคือ Abuse?

Abuse คือการใช้ subdomain บน [*].id.thatako.net ในทางที่ละเมิด ข้อกำหนดการใช้งาน หรือก่อให้เกิดความเสียหายต่อผู้อื่นหรือต่อชื่อเสียงของ thatako.net

  • ประเภทที่ควรรายงาน
  • Phishing เว็บไซต์ที่ออกแบบมาเพื่อขโมยข้อมูล รหัสผ่าน หรือข้อมูลส่วนตัว
  • Malware / Exploit เว็บไซต์ที่แจกจ่ายไฟล์อันตราย หรือโจมตีผู้เข้าชม
  • เนื้อหา NSFW สื่อลามก หรือเนื้อหาที่ไม่เหมาะสมสำหรับผู้เยาว์
  • เนื้อหาผิดกฎหมาย เนื้อหาที่ผิดกฎหมายไทย ละเมิดลิขสิทธิ์ หรือหมิ่นประมาท
  • ปลอมแปลงเป็นโรงเรียน เว็บที่แสร้งว่าเป็นเว็บทางการของโรงเรียนท่าตะโกพิทยาคม หรือบุคลากร
  • Spam / Scam เว็บไซต์โฆษณาหรือหลอกลวงในเชิงพาณิชย์

ช่องทางที่ 2 - อีเมล

ส่งอีเมลมาที่ aitji@duck.com ใช้ subject: Abuse Report [ชื่อ subdomain] และระบุข้อมูลเดียวกับ GitHub Issues ด้านบน

รายงานที่มีหลักฐาน (archive.org, screenshot, URL) จะถูกดำเนินการเร็วกว่ารายงานที่ไม่มีหลักฐาน
maintainer จะพยายามตอบกลับและดำเนินการให้เร็วที่สุด นอกเรียน/วันหยุดนักขัตฤกษ์

กระบวนการตรวจสอบ

  • 1
    รับรายงาน maintainer ได้รับรายงานทาง GitHub Issues หรืออีเมล
  • 2
    ตรวจสอบ maintainer เข้าตรวจสอบ subdomain และหลักฐานที่ได้รับ
  • 3
    แจ้ง owner หาก abuse ไม่ชัดเจน maintainer จะ tag owner ให้ชี้แจง ภายในระยะเวลาที่กำหนด
  • 4
    ดำเนินการ หากยืนยันว่าเป็น abuse จะระงับ subdomain ทันที หรือออก warning กรณีเป็นครั้งแรกและความรุนแรงต่ำ
  • 5
    แจ้งผล maintainer จะอัปเดตผลการดำเนินการใน Issue หรือตอบกลับอีเมล

    • กรณี Phishing, Malware หรือ NSFW ที่ชัดเจน subdomain จะถูกระงับทันที โดยไม่ต้องรอการชี้แจงจาก owner

รายงาน Subdomain ของตัวเองที่ถูกระงับ

หากเชื่อว่า subdomain ของตัวเองถูกระงับโดยไม่มีเหตุผล สามารถอุทธรณ์ได้โดย ติดต่อ maintainer พร้อมระบุ subdomain และเหตุผลที่คิดว่าเป็นการเข้าใจผิด
